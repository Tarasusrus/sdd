Role

You are a Software Architect defining technical constraints for an AI coding agent.

Project Context

See Tech Stack section in @file:proposal.md

Codebase facts:
- Request flow: HTTP Handler (internal/server/) → Service (internal/service/) → Repository (internal/service/repository/) → Database (internal/database/)
- Groups: course_group table, UpdateCourseGroup service handles adding/removing students
- Async enrollment: AddStudentToLastGroupByApplicationId assigns student to last created group
- Materials: GetMaterialsBySuser resolves groupId internally but does NOT expose group type in response
- model.Material has GroupId *int but no group type field
- Mocks: generated by mockery from internal/service/repository/ and internal/service/interfaces/
- Tests: internal/tests/server/ and internal/tests/service/, testify + table-driven
- Logging: h.logger.LogError(ctx, err, loglib.ErrorMeta{Type, Code}) — see CLAUDE.md

Task

Analyze @file:requirements.md and @file:acceptance_criteria.md and define technical constraints covering:
- DB schema changes (what tables/columns to add or reuse)
- Component design (new services, interfaces, repositories)
- API design (new endpoints, changes to existing responses)
- Code style (naming, patterns to follow, anti-patterns to avoid)
- Testing strategy (what to test, how)
- Decisions & Trade-offs — for every non-obvious choice: what was picked, what alternatives were rejected, why. This section is mandatory. A year from now this is the most valuable part of the spec.

Output Format

Use clear imperative statements (MUST / SHOULD / MUST NOT).
The agent must be able to validate its implementation against each constraint.

Output File

Write the results to spec/constraints.md
